# Apple-Sales-Analysis
Apple Store Data Analysis using PostgreSQL and SQL. This project includes data exploration, filtering, sorting, joins, aggregations, GROUP BY, subqueries, revenue analysis, sales analysis, warranty claim analysis, and business insights generation from Apple Store datasets.

📊 Apple Store Data Analysis

This project analyzes Apple Store datasets using SQL and PostgreSQL.

Key Concepts Used:
- SELECT, WHERE, ORDER BY
- Aggregate Functions (SUM, COUNT, AVG)
- GROUP BY and HAVING
- INNER JOIN and LEFT JOIN
- Subqueries
- Sales Analysis
- Product Analysis
- Store Performance Analysis
- Warranty Claims Analysis
- Revenue Calculation

The goal of this project is to practice real-world SQL queries and generate business insights from Apple Store data.


🔹 Grouping & Aggregation Analysis
Total Sales by Each Store

Analyzed store-wise sales performance by grouping sales records and calculating total transactions for each store.

Warranty Claims by Repair Status

Grouped warranty claims based on repair status to identify the frequency of different claim outcomes.

Total Units Sold by Product Category

Calculated category-wise units sold by joining sales and product datasets.

Product Count by Category

Counted the number of products available in each category and sorted categories numerically.

Revenue Generated by Each Product (Last Year)

Used aggregate functions and date filtering to calculate product-wise revenue for the most recent one-year period.

🔹 Multi-Table Joins
Sales with Product Information

Joined sales and product tables to combine transaction details with product information.

Warranty Claims Related to Sales

Connected warranty claims with sales records using sale identifiers.

Products with Categories

Retrieved product information along with their respective categories.

Sales with Store Information

Joined sales and store tables to analyze transactions at the store level.

Warranty Claims with Product Names

Used multiple joins across warranty, sales, and product tables to display warranty claims alongside corresponding product names.

🔹 Subqueries & Advanced SQL
Unsold Products Analysis

Identified products that have never been sold using subquery and LEFT JOIN techniques.
